# Election Session Creation

The **Election Session Creation** module is where organization officers set up new voting events. To ensure platform sustainability, the process includes a mandatory payment phase. After defining the election title and schedule, officers must submit proof of payment (e.g., a transaction receipt). A **Superadmin** then verifies the payment to move the election from a "**Draft/Pending**" state to a "**Scheduled**" state. This module ensures that every election on the platform is officially sanctioned and funded before it goes live.

# Use Case Scenario

| Field                                 | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| :------------------------------------ | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Use Case Name**                     | Create & Pay New Election Session                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **Actor**                             | System Admin / Election Officer & Superadmin                                                                                                                                                                                                                                                                                                                                                                                                                     |
| **Pre-conditions**                    | 1. Officer is logged in and belongs to an approved organization.<br>2. Position templates or custom positions are defined.                                                                                                                                                                                                                                                                                                                                       |
| **Trigger**                           | Admin clicks the **"Create New Election"** button in the Elections Management section.                                                                                                                                                                                                                                                                                                                                                                           |
| **Main Success Scenario**             | 1. Officer enters the **Election Title** and sets the **Start/End Dates**.</br> 2. System creates the session in '**draft**' status. </br> 3. Officer is prompted to the **Payment Gateway** to upload a receipt/screenshot. </br>4. System records the transaction in the **election_payments** ledger. </br>5. **Superadmin** reviews the receipt and clicks "**Verify**". </br>6. System updates election status to '**scheduled**' and notifies the officer. |
| **Alternative Flow (Cancel)**         | 1. Admin clicks the **"Cancel"** button.<br>2. System closes the window without saving any changes.                                                                                                                                                                                                                                                                                                                                                              |
| **Exception Flow (Empty Fields)**     | 1. Admin tries to create the election while leaving a field blank.<br>2. System shows an alert asking the user to fill in all required information before proceeding.                                                                                                                                                                                                                                                                                            |
| **Exception Flow (Missing Receipt)**  | 1. Officer fills out the election details but forgets to upload the receipt and clicks submit.<br>2. System blocks the submission and highlights the payment upload field, requiring the file before proceeding.                                                                                                                                                                                                                                                 |
| **Exception Flow (Rejected Payment)** | 1. Superadmin finds the uploaded receipt is invalid or blurred.</br>2. Superadmin sets payment status to '**rejected**'.</br>3. Officer is notified to upload a valid receipt before the draft can be scheduled.                                                                                                                                                                                                                                                 |
| **Post-conditions**                   | The election session is fully paid for, verified by a Superadmin, and officially scheduled.                                                                                                                                                                                                                                                                                                                                                                      |
